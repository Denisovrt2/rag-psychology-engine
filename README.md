# RAG Psychology Engine

## Что это

RAG Psychology Engine — AI-система для поиска и генерации ответов на основе большого корпуса специализированных текстов.

Проект решает прикладную задачу:
не просто “отвечать как LLM”, а находить релевантные фрагменты в большом массиве источников и строить ответ на основе retrieval pipeline.

## Масштаб

В рабочем контуре проекта:
- обработано и векторизовано 1000+ книг и материалов
- построен ingestion pipeline для подготовки корпуса
- реализован retrieval-подход для работы с большим массивом текстов
- AI-слой используется поверх поиска, а не вместо него

## Задача системы

Проблема обычных LLM без retrieval:
- галлюцинации
- общий ответ без опоры на источник
- потеря точности в узкой предметной области

Этот проект строится вокруг другой логики:

- сначала извлечь релевантные фрагменты
- затем уменьшить шум
- потом собрать ответ поверх найденных источников
- сделать поведение системы устойчивым к разным формулировкам запроса

## Pipeline

1. Парсинг документов  
2. OCR для сложных источников  
3. Очистка текста  
4. Перевод части корпуса через API  
5. Нормализация  
6. Чанкинг  
7. Векторизация  
8. Retrieval  
9. Reranking  
10. Генерация ответа  

## Почему этот проект сильный

Это не “чат по документам”.

Сильная сторона проекта:
- работа с большим корпусом
- data pipeline до AI-слоя
- борьба с шумом retrieval
- фокус на качестве evidence, а не только на генерации

Ключевая идея:
**качество итогового ответа определяется качеством retrieval и подготовки корпуса.**

## Ключевой инженерный артефакт

В репозитории добавлен публичный фрагмент логики подготовки корпуса под retrieval:

`src/pipeline/chunking_real_demo.py`

Этот файл показывает, что качество RAG начинается не с модели, а с правильной подготовки текста:

- нормализация сырого текста
- устойчивое разбиение PDF-подобных источников
- переход от сырого документа к смысловым chunks
- подготовка корпуса под retrieval и downstream AI-ответ

Ключевая идея:
**chunking — это не техническая мелочь, а один из главных факторов качества retrieval.**

Именно поэтому в проекте важен не только LLM-слой, но и data pipeline до него:
raw document → normalization → paragraph logic → chunks → retrieval → answer
